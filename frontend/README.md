# Frontend

学生成长诊断与路径规划页面，由 Go 后端托管静态资源，并通过 SSE 接收异步诊断事件。当前诊断必须通过 Legato 后端完成简历解析；岗位推荐、匹配差距和目标岗位雷达由 Benchmark 后的 Legato Job Matching Team via Presto 生成。该 Team 先由 Planner 动态派生多视角 Agent，再由 Synthesis Arbiter 综合输出，前端 chat 以结构化状态卡展示流式进度。路径规划仍为模拟数据，并在页面中可见标记。

## 页面能力

- 简历必传，上传后解锁向下滑动。
- 成绩单和其他材料可选。
- 通过 SSE 接收异步诊断事件，按 Agent 返回结果逐步填充模块。
- 展示能力画像、证据评分、雷达图数据、TOP5 匹配岗位。
- 底部进度条按 Agent 状态从处理中变为完成，并逐步解锁画像、匹配、路径、输出模块。
- 展示岗位匹配度、个人/岗位目标雷达、差距明细、推荐理由和可视化报表。
- 展示 30、60、90 天成长路径，包含阶段目标、周任务、资源链接和达标标准。
- 支持能力画像 JSON、Excel，路径规划 PDF、Word，匹配报表 JSON。
- 当 Legato 后端不可用或必需解析失败时，前端显示失败状态，不再回退到本地模拟诊断。

## 运行

前端由 `../backend` 的 Go 服务托管：

```sh
cd ../backend
go run .
```

打开 `http://127.0.0.1:8090`。
